# short-drama-writer: AI短剧/短视频生成 Claude 插件设计

> 方案B：Claude Code 插件 + 轻量后端 | 2026-06-03 | 探索产物

## 1. 背景与动机

现有 xiaoshuo_project 通过 Python Agent 编码实现创意管线，Agent 能力受限于固定 prompt 模板。借鉴 webnovel-writer 插件模式（skills + agents + knowledge base + data chain），用 Claude Code 插件替代 Python Agent 层，让 Claude 直接做创意决策。

**核心思路：** 插件负责"创作脑"，轻量后端负责"基础设施手"。

**与现有项目关系：** 新项目独立建设，零代码复用。数据模型全新设计，不兼容现有结构。后续可复用现有后端的 API 提供者适配器模式。

**职责边界：**
- 插件：所有创意决策、剧本生成、分镜设计、质量审查、跨集连贯性追踪、增强提示词构造
- 后端：图像/视频 API 代理（Jimeng 4.6/MiniMax/Doubao Seedance）、MySQL 索引与搜索、导出格式化、仪表盘 API、2D 线框图渲染

## 3. Skills（7个）

| Skill | 命令 | 职责 | 门控 |
|-------|------|------|------|
| `drama-init` | 项目初始化 | 收集选题方向→生成世界观→角色档案→视觉风格→量级/平台定位 | 6项前置条件通过后方可执行生成 |
| `drama-plan` | 分集规划 | 基于总纲生成分集大纲（情绪曲线、节拍、每集梗概），增量写回设定 | 大纲完整性校验 |
| `drama-write` | 单集剧本 | 上下文研究(context-agent)→起草(场景+对白+动作+取景框+分镜流图+ASCII布局)→审查(reviewer)→润色→提交(data-agent) | 审查阻塞项清零方可提交 |
| `drama-review` | 质量审查 | 设定一致/连续集接续/角色一致/AI味检测/分镜可行性/镜头连续性 | 无 |
| `drama-generate` | 资产生成 | 读取提示词JSON→调用后端API生成角色图/场景图/视频 | 无 |
| `drama-query` | 项目查询 | 查询角色/场景/风格设定/跨集状态/伏笔状态 | 无 |
| `drama-dashboard` | 仪表盘 | 启动只读Web面板，查看项目全貌、视频预览、素材库、角色图谱 | 无 |

**管线顺序：** `init → plan → write → generate`，`review` 和 `query` 辅助支撑。

**交互模式：** 斜杠命令驱动创作流程，Claude 对话负责具体创作和迭代。

## 4. Agents（4个）

每个 Agent 只输出结构化 JSON，不直接写状态。状态写入由 skill 主流程 + Python 脚本完成。

| Agent | 触发时机 | 输入 | 输出 |
|-------|---------|------|------|
| **context-agent** | 每集剧本写作前 | 设定文件 + 前集摘要 + 记忆/ + 分集大纲 | 剧本任务书（5段：前情回顾/本集目标/出场角色状态+情感锚点/取景框策略+禁忌池/结尾钩子） |
| **reviewer** | 剧本起草后 | 剧本全文 + 角色卡 + 场景卡 + 分镜流图 | 结构化问题清单 JSON（severity/category/evidence/fix_hint/blocking），不做主观评分 |
| **data-agent** | 剧本定稿后 | 定稿剧本 | 状态增量 JSON（角色状态变化/场景变化/道具服装/伏笔/事件），写回记忆/目录 |
| **deconstruction-agent** | init阶段（可选） | 用户提供的参考短剧/电影文本 | 抽象模式 JSON（分镜模式/节奏模板/爽点循环/角色互动），禁止污染项目设定 |

**reviewer 检查维度（7项）：**

| 维度 | 检查内容 |
|------|---------|
| setting | 角色能力是否匹配设定？场景描述是否违反世界规则？ |
| timeline | 时间是否衔接前集？同一角色是否同时出现在两个地点？ |
| continuity | 前集钩子是否回应？场景过渡是否合理？情绪弧线是否连续？ |
| character | 对白是否符合人设？行为是否符合动机？认知边界是否正确？ |
| logic | 因果关系是否成立？决策是否有动机支撑？ |
| ai_flavor | 5子维度：词汇/句式/叙事/情感/对白 |
| shot_continuity | 人物位置是否物理合理？服装是否无故变化？朝向关系是否匹配？180度法则？道具穿帮？ |

## 7. 角色 & 场景一致性体系

四层防御：

**第一层：生成前锚定（drama-init）**
- 角色卡含：外貌锚点、性格锚点、禁忌行为、角色弧
- 场景卡含：空间锚点、氛围约束、道具清单

**第二层：生成中约束（context-agent → 剧本任务书）**
- 本集出场角色 × 当前状态 × 被禁止的行为
- 本集场景 × 空间锚点 × 允许的取景框范围
- 上一集结束时的未闭合伏笔（必须在本集处理或推进）

**第三层：生成后校验（reviewer + data-agent）**
- reviewer 检查角色/场景一致性 + 镜头连续性
- data-agent 提取本集变化，写回记忆/

**第四层：跨集闭环（记忆/目录）**
- 角色状态变迁.json：每集结束时的角色状态快照
- 场景状态.json：每集结束时的场景变化
- 伏笔追踪.json：已埋/已回收/待回收
- 道具-服装追踪.json：跨集道具/服装一致性

## 8. 分镜流图（Shot Flow Map）

`drama-write` 生成每个场景时，额外产出分镜流图 JSON，追踪镜头间空间和外观连续性：

```json
{
  "scene_id": "S03",
  "scene_name": "咖啡厅对峙",
  "shots": [
    {
      "shot_id": "S03_F01",
      "frame_type": "中景",
      "characters": [
        {
          "name": "林峰",
          "position": "门口站立",
          "facing": "面向咖啡厅内部",
          "outfit": "黑色西装"
        }
      ],
      "notes": "开场建立镜头"
    },
    {
      "shot_id": "S03_F02",
      "frame_type": "近景",
      "characters": [
        {
          "name": "林峰",
          "position_change": "门→吧台前（3步）",
          "current_position": "吧台前",
          "outfit": "黑色西装（不变）"
        }
      ],
      "continuity": "接S03_F01，林峰推门后向前走"
    }
  ],
  "ascii_layout": "┌──────────────────────┐\n│     [吧台]            │\n│  A(门)               │\n│              C1      │\n│    B                 │\n│  [窗边座位区]         │\n└──────────────────────┘\nA=林峰 B=陈世荣 C1=客人"
}
```

**配套视觉锚点（渐进落地）：**
- MVP：ASCII 空间布局 + 分镜流图（纯文本），作为 reviewer 检查输入
- V1：后端将 ASCII 渲染为 2D 线框俯视图/立面图，仪表盘展示，用户可确认
- V2：线框图嵌入视频 API 作为多模态参考（视平台支持情况）

## 10. 轻量后端

**技术栈：** Python FastAPI + MySQL + 文件存储 + TOS（火山引擎对象存储）

**核心模块：**

| 模块 | 职责 |
|------|------|
| API 代理层 | 封装 Jimeng 4.6 / MiniMax / Doubao Seedance。统一入参适配。并发限流（窗口1） |
| 项目管理 API | 项目 CRUD、剧集 CRUD、MySQL 索引 |
| 素材管理 API | 素材上传/查询/分组/版本管理。存储到本地 + TOS |
| 导出模块 | 分镜提示词格式化为各平台格式（Jimeng/Sora/Runway/Seedance） |
| 仪表盘 API | 项目状态、剧集列表、素材预览、角色图谱 |
| 布局渲染器（V1） | ASCII 布局 → 2D 线框俯视图/立面图 |

**关键端点：**
```
POST /api/generate/image      → 生成角色/场景参考图
POST /api/generate/video      → 生成单镜头视频
GET  /api/generate/status/{id} → 查询生成任务状态
GET  /api/projects            → 项目列表
POST /api/projects            → 创建项目
GET  /api/projects/{id}       → 项目详情
GET  /api/episodes/{id}       → 剧集详情（剧本+分镜+提示词）
POST /api/assets/query        → 素材搜索/过滤
POST /api/export/prompts      → 批量格式化提示词
```

## 11. 用户交互

- **Claude 对话：** 所有创意和生产——选题、大纲、剧本、提示词、触发生成。`/drama-*` 命令驱动。
