## 固定物空间布局（影响所有取景框 → 强制贯穿视频提示词）

| 固定物 ID | 名称 | 画面位置 | 大小 | 朝向 | 备注 |
|----------|------|---------|------|------|------|
| | | 前/左/右/中央/后景 + 区域 | 占画面比例 | 朝向描述 | |

**说明**：
- ID 用英文短码（下划线分隔），全剧统一。如 `sofa_L`、`window_R`、`tv_wall`
- 画面位置用结构化方向：前(后景)/左/右/中央/上(天花)/下(地面)
- 朝向：物体开口/正面/背面的指向。如沙发"开口朝右"、电视墙"正面朝镜头"
- 至少列出 3-5 个本场景的主要固定物
- 禁止写通用物件（"墙"、 "地"）作为固定物，只写可作为镜头参照的实体物体
- 写入 scene-master --layout 时，转换为如下 JSON：
```json
{
  "view": "front_elev",
  "fixed_objects": [
    {"id": "<ID>", "name": "<名称>", "position": "<位置>", "size": "<大小>", "orientation": "<朝向>"}
  ],
  "walkable_zones": [
    {"id": "<zone_id>", "position": "<位置>", "description": "<说明>"}
  ]
}
```

## 取景框定义（按需生成，不在初始化时填写）

取景框在 drama-write 阶段根据剧本实际需要动态定义，而非预设。

| 取景框ID | 类型 | 触发条件 | 景别 | 机位描述 | 焦点区域 |
|----------|------|---------|------|---------|---------|
| {场景}_master | 全景主图 | 本场景首次出场 | wide/establishing | 场景正前方 | 完整空间 |
| {场景}_{区域}_2shot | 中景双人 | 两人同框对话/冲突 | medium | 核心活动区正面 | 吧台/沙发等核心区域 |
| {场景}_{区域}_cu | 近景特写 | 单人情绪/台词爆点 | close-up | 距人物1-2米 | 面部区域(背景虚化) |
| {场景}_{区域}_os | 过肩视角 | 一方质问/试探 | medium-close | 从A背后拍向B | A肩前景虚化,B面部清晰 |
| {场景}_{物体}_ecu | 大特写 | 关键道具揭示 | extreme-cu | 垂直俯拍 | 单一物体 |

取景框生成时全部为空镜（无人物），以场景全景主图为 reference_image 保证空间一致性。
